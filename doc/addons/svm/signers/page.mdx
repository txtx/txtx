# Solana and SVM Compatible Blockchains (beta) Signers {{ id: "svm-signers" }}



## secret_key {{ tag: 'signer' }}

The `svm::secret_key` signer can be used to synchronously sign a transaction.

### Inputs

<Properties>


  <Property name="secret_key" required="optional" type="string">
    The secret key used to sign messages and transactions.
  </Property>


  <Property name="mnemonic" required="optional" type="string">
    The mnemonic phrase used to generate the secret key. This input will not be used if the `secret_key` input is provided.
  </Property>


  <Property name="derivation_path" required="optional" type="string">
    The derivation path used to generate the secret key. This input will not be used if the `secret_key` input is provided.
  </Property>


  <Property name="keypair_json" required="optional" type="string">
    A path to a keypair.json file containing the secret key. This input will not be used if the `secret_key` or `mnemonic` inputs are provided.
  </Property>


  <Property name="is_encrypted" required="optional" type="bool">
    Coming soon
  </Property>


  <Property name="password" required="optional" type="string">
    Coming soon
  </Property>



</Properties>

### Outputs

<Note>

When the `secret_key` action is successfully executed, the following outputs are attached to the action

</Note>

<Properties>


  <Property name="public_key" type="string">
    The public key of the account generated from the secret key, mnemonic, or keypair file.
  </Property>


  <Property name="address" type="string">
    The SVM address generated from the secret key, mnemonic, or keypair file. This is an alias for the `public_key` output.
  </Property>


</Properties>

<CodeGroup title="Example using secret_key">


```hcl {{ title: 'main.tx' }}
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}
```

## web_wallet {{ tag: 'signer' }}

The `svm::web_wallet` signer will allow a Runbook operator to sign the transaction with the browser signer of their choice.

### Inputs

<Properties>


  <Property name="expected_address" required="optional" type="string">
    The SVM address that is expected to connect to the Runbook execution. Omitting this field will allow any address to be used for this signer.
  </Property>



</Properties>

### Outputs

<Note>

When the `web_wallet` action is successfully executed, the following outputs are attached to the action

</Note>

<Properties>


  <Property name="address" type="string">
    The address of the account. This is an alias for the `public_key` output.
  </Property>


  <Property name="public_key" type="string">
    The address of the account.
  </Property>


</Properties>

<CodeGroup title="Example using web_wallet">


```hcl {{ title: 'main.tx' }}
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}

```

## squads {{ tag: 'signer' }}

The `svm::squads` signer can be used to sign a transaction with a squads multisig.

### Inputs

<Properties>


  <Property name="multisig_account_public_key" required="optional" type="addon(svm::pubkey)">
    The Squad multisig account pubkey. This is found on the settings page in the Squads app. This is not the vault address. Rather, this multisig account address will be used to derive the vault address and all transaction PDAs.
  </Property>


  <Property name="create_key" required="optional" type="addon(svm::pubkey)">
    The create key used to derive the Squad multisig address.
  </Property>


  <Property name="vault_index" required="optional" type="integer">
    The index of the vault to be created. If omitted, `0` will be used.
  </Property>


  <Property name="initiator" required="required" type="string">
    A reference to a signer construct, which will be used to create the Squads Vault Transaction & Proposal. This signer must have the `Initiate` permission in the Squads Multisig.
  </Property>


  <Property name="payer" required="optional" type="string">
    A reference to a signer construct, which will be used to pay for the Squads Vault Transaction & Proposal creation. If omitted, the `initiator` will be used.
  </Property>


  <Property name="program_id" required="optional" type="addon(svm::pubkey)">
    The program ID of the Squad program. If omitted, the default program ID will be used.
  </Property>


  <Property name="squads_frontend_url" required="optional" type="string">
    The URL of the Squads frontend. If omitted, the default URL 'https://app.squads.so' will be used.
  </Property>



</Properties>

### Outputs

<Note>

When the `squads` action is successfully executed, the following outputs are attached to the action

</Note>

<Properties>


  <Property name="public_key" type="string">
    The public key of the Squad vault for the provided vault index. This is an alias for the `vault_public_key` output
  </Property>


  <Property name="address" type="string">
    The public key of the Squad vault for the provided vault index. This is an alias for the `vault_public_key` output
  </Property>


  <Property name="vault_public_key" type="string">
    The public key of the Squad vault for the provided vault index.
  </Property>


  <Property name="vault_address" type="string">
    The public key of the Squad vault for the provided vault index. This is an alias for the `vault_public_key` output
  </Property>


  <Property name="multisig_account_public_key" type="string">
    The public key of the Squad multisig pda. This address should not be funded.
  </Property>


  <Property name="multisig_account_address" type="string">
    The public key of the Squad multisig pda. This address should not be funded. This is an alias for the `multisig_account_public_key` output
  </Property>


</Properties>

<CodeGroup title="Example using squads">


```hcl {{ title: 'main.tx' }}
signer "initiator" "svm::web_wallet" {
    expected_address = input.initiator_address
}
signer "deployer" "svm::squads" {
    multisig_account_public_key = input.squads_multisig_address
    initiator = signer.initiator
}

```
