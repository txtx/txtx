# Solana and SVM Compatible Blockchains (beta) Signers {{ id: "svm-signers" }}



## secret_key {{ tag: 'signer' }}

The `svm::secret_key` signer can be used to synchronously sign a transaction.

### Inputs

<Properties>


  <Property name="secret_key" required="optional" type="string">
    The secret key used to sign messages and transactions.
  </Property>


  <Property name="mnemonic" required="optional" type="string">
    The mnemonic phrase used to generate the secret key. This input will not be used if the `secret_key` input is provided.
  </Property>


  <Property name="derivation_path" required="optional" type="string">
    The derivation path used to generate the secret key. This input will not be used if the `secret_key` input is provided.
  </Property>


  <Property name="keypair_json" required="optional" type="string">
    A path to a keypair.json file containing the secret key. This input will not be used if the `secret_key` or `mnemonic` inputs are provided.
  </Property>


  <Property name="is_encrypted" required="optional" type="bool">
    Coming soon
  </Property>


  <Property name="password" required="optional" type="string">
    Coming soon
  </Property>



</Properties>

### Outputs

<Note>

When the `secret_key` action is successfully executed, the following outputs are attached to the action

</Note>

<Properties>


  <Property name="public_key" type="string">
    The public key of the account generated from the secret key, mnemonic, or keypair file.
  </Property>


  <Property name="address" type="string">
    The SVM address generated from the secret key, mnemonic, or keypair file. This is an alias for the `public_key` output.
  </Property>


</Properties>

<CodeGroup title="Example using secret_key">


```hcl {{ title: 'main.tx' }}
signer "deployer" "svm::secret_key" {
    secret_key = input.secret_key
}

```

## web_wallet {{ tag: 'signer' }}

The `svm::web_wallet` signer will allow a Runbook operator to sign the transaction with the browser signer of their choice.

### Inputs

<Properties>


  <Property name="expected_address" required="optional" type="string">
    The SVM address that is expected to connect to the Runbook execution. Omitting this field will allow any address to be used for this signer.
  </Property>



</Properties>

### Outputs

<Note>

When the `web_wallet` action is successfully executed, the following outputs are attached to the action

</Note>

<Properties>


  <Property name="address" type="string">
    The address of the account.
  </Property>


  <Property name="public_key" type="string">
    The address of the account.
  </Property>


</Properties>

<CodeGroup title="Example using web_wallet">


```hcl {{ title: 'main.tx' }}
signer "alice" "svm::web_wallet" {
    expected_address = "zbBjhHwuqyKMmz8ber5oUtJJ3ZV4B6ePmANfGyKzVGV"
}

```

## squads {{ tag: 'signer' }}

The `svm::squads` signer can be used to synchronously sign a transaction.

### Inputs

<Properties>


  <Property name="address" required="optional" type="string">
    The squad vault address.
  </Property>



</Properties>

### Outputs

<Note>

When the `squads` action is successfully executed, the following outputs are attached to the action

</Note>

<Properties>


  <Property name="public_key" type="string">
    The public key of the squad vault.
  </Property>


  <Property name="address" type="string">
    The SVM address generated from the secret key, mnemonic, or keypair file. This is an alias for the `public_key` output.

</Properties>
